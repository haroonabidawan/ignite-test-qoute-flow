# Internationalization

English and Arabic support via `@repo/i18n` - **no `/en` or `/ar` in routes**.

## Usage

```tsx
import { useTranslation } from "@repo/i18n";

const { t, locale, setLocale, dir } = useTranslation();
t("quotations.create.title");
t("quotations.create.ai.usageSummary", { hourlyRemaining: 5, hourlyLimit: 10 });
```

## Locale storage

- Key: `quoteflow_locale` in `localStorage`
- Fallback: browser language (`ar*` → Arabic, else English)
- `document.documentElement.lang` and `.dir` updated on change

## Adding strings

Edit both catalogs:

- `packages/i18n/src/locales/en.ts`
- `packages/i18n/src/locales/ar.ts`

API error codes map to `errors.{code}` (e.g. `errors.ai_rate_limited`).

## RTL

- Arabic sets `dir="rtl"` on `<html>`
- Use logical CSS where possible (`ms-`, `me-`, `start`, `end`)
- Arabic font: Noto Sans Arabic (loaded in web layout)

## Language switcher

`LanguageSwitcher` in `@repo/ui` - compact **EN** / **AR** buttons in auth and app shells.

## AI locale

`POST /quotations/ai-draft` accepts `locale: "en" | "ar"` - backend instructs the model which language to write titles/descriptions in.
